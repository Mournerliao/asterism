# 0023 · 批量关系写入使用持久化逐项执行账本

- Status: Accepted
- Date: 2026-07-19

Phase 2 先建立不依赖 AI 的可靠批量整理路径。用户确认后，把选择范围固化为 repository ID 集合，并把每条“仓库 × 标签或集合 × 添加或移除动作”持久化为独立执行项目；成功项目立即保留，失败项目分为可重试与终止，只重试可重试项目，添加已有关系或移除不存在关系视为幂等成功。相比单次前端请求或整批事务，这会增加操作与项目状态的存储及清理成本，但能在网络中断、页面刷新或部分失败后准确恢复，也让人工批量操作和 AI 草稿确认共用同一条可审计写入路径，而不产生两套可靠性语义。
