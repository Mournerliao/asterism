# 2026-07-13 · 导入 / 导出交互澄清

## 目标

消除导出区“格式按钮与下载按钮分别代表什么”的理解成本，让用户无需试错即可判断三种格式的用途、数据范围与恢复能力。

## 变更

- 将格式选择 + 通用 Download 的隐含两步操作改为 JSON / CSV / Markdown 三条直接下载操作。
- 为每种格式补充用途与数据范围：JSON 是可恢复的完整备份；CSV 是仅含仓库元数据的表格清单；Markdown 是按集合、标签与笔记组织的可读归档。
- JSON 使用主按钮表达推荐的备份路径，CSV / Markdown 使用次级按钮；按钮文案直接包含目标格式。
- 移除原始文本导出预览，避免大段机器可读内容与主要任务争夺注意力。
- 导入区改称“恢复备份”，明确仅作用于已同步仓库，不会新增 star 或仓库。
- 同步更新 en / zh-CN i18n 文案，并修正产品契约中将三种格式均描述为可导入的历史偏差。

## 设计与实现约束

- 复用现有 Button、Graphite Glass tokens、8px 容器圆角与 4px 间距体系，没有新增设计 token 或依赖。
- 导出格式使用同一实体内容面板内的分隔列表，不创建嵌套卡片。
- 每个下载按钮通过 `aria-describedby` 关联对应格式说明；窄屏下按钮占满行宽，桌面下与说明同行。
