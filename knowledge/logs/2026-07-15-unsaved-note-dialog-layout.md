# 未保存笔记确认弹窗布局修正

日期：2026-07-15

## 问题

确认弹窗直接使用通用 `512px / 24px` 密度，短内容显得松散；Footer 又通过 `justify-between` 与嵌套 flex 把「继续编辑」和另外两个动作拆开，三个互斥决策没有形成稳定操作组。

## 调整

- 局部覆盖为最大宽度 `448px`、四边 `20px`，标题与说明保持 `8px` 紧密分组，说明到操作区使用 `20px` 分隔。
- 移除 Footer 的嵌套分组和 `space-between`，三个按钮成为同层直接子项，并按继续编辑 → 放弃并继续 → 保存并继续右对齐。
- 移除动作文案中重复的「并继续」，并把说明收敛为单句，避免英文最小内容宽度撑破 Grid、吞掉右侧 padding。
- 三个按钮统一使用紧凑尺寸；窄屏改为同宽单列，主操作位于阅读流末端。
- 不修改共享 Dialog 默认值，避免影响标签、集合和通用确认弹窗。

## 验证

- 组件约束为 Dialog 最大宽度 `448px`、四边 padding `20px`、Header 到 Footer gap `20px`；缩短后的三个动作可在同一 32px 基线上收拢于右侧，不再撑破内容轨道。
- `<640px` 时三个按钮使用同宽单列，Dialog 沿用全局 `calc(100% - 2rem)` 安全边距，无横向溢出。
- Impeccable layout detector 无命中；lint、Web typecheck 与 28 项 Web 测试通过。
